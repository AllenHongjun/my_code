1) token:

用户登录后会得到sessionid, 之后请求的url添加参数skey={sessionid} 如
http://localhost/member/get?skey=1657ab575c8c470f85911c0a766293cf

2) 用户Id:【每个请求须传递该参数,没有时传0】

用户登录后会得到uid, 之后请求的url添加参数uid={userid} 如
http://localhost/member/get?uid=412223

3) rt(待定):

不同的请求类型rt={requesttype}，三个值中取一个weixin,app,web 如
http://localhost/member/get?rt=weixin

4) v:

不同的包带过来的版本参数不一样，线上问题定位时用
http://localhost/member/get?v=v1.0

5) 返回值说明： 

{
	state:1,
	resultcode:200,
	errordes:"错误说明（用于用户显示的说明文字）",
	result:{}
}
state：成功为1，失败为0
resultcode：
错误代码 详细描述
200 成功
403 未登录
404 接口不存在
500 未知错误
503 已知错误
1000以上 接口特定错误
result数据内容说明：
1. 必须是一个{}对象
2. 如果是文字内容包含属性info值
举例：
result:
{
	info:"你已经添加手机号码"
}
3. 如果是数据列表在对象内包含list属性，值是一个数组，如果包含分页信息放在外层
举例：
result:
{
	info:"商品列表",
	size:12,
	page:2,
	total:120,
	list:[
	{
		id:1,
		title:"道具A"
	},
	{
		id:12,
		title:"道具B"
	}]
}
